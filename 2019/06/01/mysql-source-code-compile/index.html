[数据库]Mac下编译、安装、调试MySQL 8.0.16

Posted on 2019-06-01 | In 数据库 | Comments: | Views:

本文由本人（Haoxiang Ma）原创，转载请注明出处。同时借鉴了几篇同类型的旧文章，已在文尾注明了详细出处。

本文写于2019/06/01。

Overview

作为数据库内核 or 存储引擎的开发者，或者对MySQL原理很感兴趣的同学，很多时候需要对源码进行debug。但是一般情况下，大家很少直接从源码出发构建自己开发环境上的MySQL，也有很多同学不知道怎么用IDE结合源码来debug一个正在运行着的应用程序（不一定是MySQL，可能是别的应用如Elasticsearch）。

截至本文发表时，MySQL最新稳定版为8.0.16。所以我们就用此版本来说明，如何在Mac环境下从MySQL源码出发，进行编译、安装、然后用Jetbrains家的IDE——CLion对MySQL作调试。

环境说明

  • MySQL 8.0.16
  • macOS Mojave 10.14.2

准备工作

  1. 确保环境中已安装好g++、cmake

  2. 下载 MySQL源码（自带boost库的版本）

    如图所示，在下载类型处选择“Source Code”，这样下载的才是源码，而不是可直接安装的安装包。

从源码编译并安装

  1. 把下载的源码压缩包解压，我们假设解压至/Users/your_name/mysql-8.0.16
  2. 确定basedir，一般就是源码所在目录，即/Users/your_name/mysql-8.0.16
  3. 确定datadir，一般是basedir下的data目录，即/Users/your_name/mysql-8.0.16/data

  4. 进入源码目录

    cd /Users/your_name/mysql-8.0.16/

  5. 通过cmake生成Makefile

    注意，因为我们下载的源码就是带boost库的，所以不用加boost库的下载参数，可以直接用本地的boost库（-DWITH_BOOST=……）。

  6. 以上cmake执行完后，会在当前目录下生成Makefile，此时可直接make

    如果你的开发机是多核的，可指定多核参数-j加速make过程。例如开发机有16核，即可：make -j 16

    否则，直接执行：make

  7. make完成后，需要make install

    同理，也可指定多核参数加速：make install -j 16，或直接make install

    Note：以上cmake、make、make install的过程，在性能一般的开发机上耗时很久。

  8. 初始化数据库

    先进入bin目录：cd /Users/your_name/mysql-8.0.16/bin

    然后运行mysqld进行初始化，要记得指定basedir和datadir参数：

    Note：此处指定--initialize-insecure是为了省略root@localhost用户的密码。我觉得只是为了debug的话，不需要用户密码验证了吧= =，当然也可以给root用户设密码，随你喜欢。

  9. 完成以上8步后，编译+安装+初始化已全部完成。

利用CLion进行debug

我们从源码出发，已经成功在开发环境上装好了MySQL 8.0.16，接下来要看看怎么利用IDE结合源码来debug。

  1. 配置CLion里的cmake

    从CLion的菜单栏进入，CLion -> Preferences -> Build, Execution, Deployment -> CMake。

    点击CLion右上角，出现下拉菜单，选择Edit Configurations，如下图所示：

    然后就会出现以下界面：

    Note：Name那一栏不能再填mysqld，应该另起一个名字。而Target和Execution要在下拉菜单里慢慢仔细看，最后选择mysqld。Program Arguments里则填入我们初始化数据库时的参数：

    至于Working Directory，应填入源码所在的目录，即/Users/your_name/mysql-8.0.16。

  2. 以上都设置好后，直接点击CLion界面右上角的debug按钮（臭虫图标），等待十来秒基本就能正常启动了。

  3. 此时可以在terminal处启动客户端，尝试连接我们已启动的MySQLServer进程

    先进入bin目录：cd /Users/your_name/mysql-8.0.16/bin

    然后尝试连接Server：./mysql -h localhost -P 3306 -u root

    Note：因为之前初始化时我设置了–initialize-insecure，所以连接时不需要指定-p参数来输入密码，否则，要加上-p并输入你给root用户设定的密码。

  4. 尽情地在CLion里给源码打断点，然后在terminal里通过客户端给Server发请求，进行debug吧~

Summary

为了要debug MySQL源码，要先做好下载、编译、安装、IDE设置。
